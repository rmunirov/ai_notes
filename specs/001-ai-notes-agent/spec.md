# Спецификация функции: AI-агент личных заметок и базы знаний

**Feature Branch**: `001-ai-notes-agent`
**Created**: 2026-04-26
**Status**: Draft
**Input**: Агент для личных заметок и базы знаний с графическим интерфейсом, семантическим поиском и диалогом на естественном языке.

---

## Пользовательские сценарии и тестирование *(обязательно)*

### Пользовательский сценарий 1 — Создание и редактирование заметки (Приоритет: P1)

Пользователь открывает приложение, нажимает «Новая заметка», вводит заголовок и текст с базовой структурой (заголовки, списки), сохраняет заметку и видит её в общем списке. При следующем запуске приложения заметка остаётся на месте.

**Почему этот приоритет**: Создание и сохранение заметок — основная ценность продукта. Все остальные функции бессмысленны без надёжного хранения контента.

**Независимый тест**: Можно полностью проверить, создав заметку, перезапустив приложение и убедившись, что заметка сохранилась. Уже на этом этапе приложение представляет ценность как простой редактор заметок.

**Сценарии приёмки**:

1. **Given** новый пользователь открывает приложение без заметок, **When** он нажимает «Новая заметка», вводит заголовок и текст, **Then** заметка появляется в списке заметок.
2. **Given** существующая заметка открыта в редакторе, **When** пользователь изменяет текст и сохраняет, **Then** изменения отражаются в списке и сохраняются при перезапуске приложения.
3. **Given** приложение не может сохранить заметку (например, нет доступа к хранилищу), **When** пользователь пытается сохранить, **Then** отображается понятное сообщение об ошибке с описанием проблемы и возможными действиями.
4. **Given** у пользователя есть несколько заметок, **When** он удаляет одну с подтверждением, **Then** заметка исчезает из списка и не восстанавливается при перезапуске.

---

### Пользовательский сценарий 2 — Семантический поиск заметок (Приоритет: P2)

Пользователь вводит запрос в поисковую строку на естественном языке. Система возвращает список релевантных заметок с фрагментами контекста. Пользователь кликает на результат и переходит к полной заметке. Поиск находит заметки по смыслу, даже если точные слова запроса в заметке не встречаются.

**Почему этот приоритет**: Семантический поиск — ключевое конкурентное преимущество над обычными заметочниками. Без него агент не может находить релевантные материалы для ответов.

**Независимый тест**: Можно проверить, создав несколько заметок на разные темы и убедившись, что запрос, сформулированный иначе, чем текст заметки, всё равно возвращает релевантные результаты.

**Сценарии приёмки**:

1. **Given** база с несколькими заметками на разные темы, **When** пользователь вводит запрос, семантически близкий к содержимому заметки (но не дословно), **Then** эта заметка появляется в результатах поиска с релевантным фрагментом.
2. **Given** запрос, не совпадающий ни с одной заметкой по смыслу, **When** пользователь отправляет поиск, **Then** отображается сообщение об отсутствии результатов (без пустого экрана без пояснений).
3. **Given** список результатов поиска, **When** пользователь кликает на результат, **Then** открывается полная заметка в редакторе.

---

### Пользовательский сценарий 3 — Диалог с агентом (Приоритет: P3)

Пользователь вводит вопрос в область диалога с агентом. Агент формирует ответ, опираясь на содержимое заметок пользователя. Ответ сопровождается ссылками на использованные заметки. Если данных недостаточно или уверенность низкая, агент прямо сообщает об этом, не выдумывая факты.

**Почему этот приоритет**: Диалог с агентом — флагманская функция продукта. Но он зависит от наличия качественного контента (P1) и работающего семантического поиска (P2).

**Независимый тест**: Можно проверить, создав несколько заметок по конкретной теме и задав связанный вопрос. Агент должен ответить точно. Затем задать вопрос вне базы — агент должен признать недостаток данных.

**Сценарии приёмки**:

1. **Given** база заметок содержит информацию, релевантную вопросу, **When** пользователь задаёт вопрос на естественном языке, **Then** агент возвращает связный ответ со ссылками на использованные заметки.
2. **Given** база заметок не содержит информации для ответа, **When** пользователь задаёт вопрос, **Then** агент явно сообщает, что в базе знаний нет достаточно релевантных данных — без домысливания.
3. **Given** информация в заметках частичная или неоднозначная, **When** пользователь задаёт вопрос, **Then** агент отвечает с явным указанием степени уверенности и не представляет предположения как факты.
4. **Given** пустая база заметок, **When** пользователь задаёт любой вопрос, **Then** агент сообщает об отсутствии заметок и предлагает создать первую.

---

### Пользовательский сценарий 4 — Онбординг нового пользователя (Приоритет: P1)

Новый пользователь запускает приложение первый раз. Без каких-либо настроек или регистрации он создаёт первую заметку и задаёт первый вопрос агенту — всё в рамках одного сеанса.

**Почему этот приоритет**: Барьер входа определяет, останется ли пользователь в продукте. Нулевая настройка критична для первого впечатления.

**Независимый тест**: Запустить приложение на чистой машине и без каких-либо предварительных действий создать заметку и задать вопрос.

**Сценарии приёмки**:

1. **Given** первый запуск приложения, **When** пользователь открывает приложение, **Then** отображается понятный пустой экран с подсказкой о том, как создать первую заметку.
2. **Given** пустое приложение, **When** пользователь создаёт первую заметку и переходит к агенту, **Then** он может задать вопрос без дополнительных настроек.

### Граничные случаи

- Что происходит при попытке сохранить заметку с пустым заголовком и телом?
- Как ведёт себя поиск при коллекции в 5000+ заметок (производительность)?
- Что видит пользователь при попытке задать вопрос, пока идёт индексация новой заметки?
- Как обрабатывается очень длинная заметка (>10 000 символов) в редакторе и при поиске?
- Что происходит, если во время ответа агента соединение с AI-сервисом прерывается?

## Требования *(обязательно)*

### Функциональные требования

- **FR-001**: Система MUST позволять создавать новые заметки с заголовком и телом.
- **FR-002**: Редактор заметок MUST поддерживать базовую структуру текста: заголовки, маркированные и нумерованные списки.
- **FR-003**: Система MUST сохранять все заметки и их изменения при штатном завершении работы приложения.
- **FR-004**: При ошибке сохранения система MUST отображать понятное сообщение с описанием проблемы и предлагаемым действием.
- **FR-005**: Пользователь MUST иметь возможность редактировать любую существующую заметку.
- **FR-006**: Пользователь MUST иметь возможность удалить заметку; удаление MUST требовать подтверждения.
- **FR-007**: Система MUST предоставлять семантический поиск по базе заметок — поиск по смыслу, а не только по точным словам.
- **FR-008**: Результаты поиска MUST отображать заголовок заметки и релевантный фрагмент текста.
- **FR-009**: Пользователь MUST иметь возможность перейти к полной заметке из результатов поиска.
- **FR-010**: Система MUST предоставлять интерфейс диалога с агентом, принимающий вопросы на естественном языке.
- **FR-011**: Ответы агента MUST формироваться исключительно на основе заметок пользователя.
- **FR-012**: Агент MUST явно сообщать о недостаточной уверенности или отсутствии релевантных данных.
- **FR-013**: Агент MUST NOT генерировать факты, не подкреплённые содержимым заметок пользователя.
- **FR-014**: Ответ агента MUST содержать ссылки на заметки, использованные при формировании ответа.
- **FR-015**: Интерфейс MUST иметь визуально разграниченные зоны: список заметок, редактор, область Q&A с агентом.
- **FR-016**: Новый пользователь MUST иметь возможность создать первую заметку и задать первый вопрос без предварительной конфигурации или регистрации.
- **FR-017**: Система MUST отображать информативные пустые состояния: когда заметок нет, когда поиск не дал результатов, когда данных для ответа недостаточно.
- **FR-018**: Пользовательские данные MUST NOT использоваться для обучения общих AI-моделей без явного согласия пользователя.

### Ключевые сущности

- **Заметка**: уникальный идентификатор, заголовок, тело (структурированный текст), дата создания, дата последнего изменения.
- **Результат поиска**: ссылка на заметку, оценка релевантности, текстовый фрагмент-превью.
- **Запрос к агенту**: текст вопроса, текст ответа, список использованных заметок, уровень уверенности, метка времени.

## Критерии успеха *(обязательно)*

### Измеримые результаты

- **SC-001**: Новый пользователь создаёт первую заметку и задаёт первый вопрос агенту в рамках одного сеанса без каких-либо предварительных настроек.
- **SC-002**: Заметки сохраняются корректно при 100% штатных завершений работы приложения.
- **SC-003**: Семантический поиск возвращает хотя бы один релевантный результат для 90% тематических запросов в коллекции из 1000+ заметок.
- **SC-004**: Агент даёт честный ответ («нет данных») в 100% случаев, когда база не содержит релевантной информации — без домысливания.
- **SC-005**: Интерфейс реагирует на действия пользователя (открытие заметки, поиск, переход между зонами) менее чем за 1 секунду при коллекции до 5000 заметок.
- **SC-006**: Семантический поиск возвращает первые результаты менее чем за 2 секунды для коллекции до 5000 заметок.
- **SC-007**: Ответ агента формируется менее чем за 5 секунд для типичного вопроса при коллекции до 5000 заметок.

## Допущения

- В первой версии поддерживается только один пользователь; мультипользовательский режим и командная работа не входят в область MVP.
- Данные хранятся локально на устройстве пользователя; облачная синхронизация не является частью MVP.
- Целевая платформа — десктопный веб-браузер или десктопное приложение; мобильные устройства и планшеты находятся вне области первой версии.
- Язык интерфейса и взаимодействия с агентом — русский по умолчанию; мультиязычность не входит в MVP.
- Редактор поддерживает богатое форматирование (rich text) в духе обычного текстового редактора — пользователю не нужно знать синтаксис разметки.
- Производительность оценивается при коллекции до 5000 коротких заметок (до ~500 слов каждая); поведение при существенно большей коллекции за пределами MVP.
- Приватность данных: система не передаёт пользовательский контент третьим сторонам для обучения моделей; это требование уровня продукта, а не только технической реализации.
